# _meta.json 配置示例

本文提供了各种场景下的 `_meta.json` 配置示例，帮助您理解如何根据不同需求配置文档结构。

## 基础示例

### 简单文档列表

最基本的配置，只包含文档标题：

```json
{
  "introduction": "介绍",
  "getting-started": "入门指南",
  "configuration": "配置",
  "usage": "使用方法",
  "faq": "常见问题"
}
```

### 带描述的文档列表

添加描述信息的配置：

```json
{
  "introduction": {
    "title": "介绍",
    "description": "项目概述和基本信息"
  },
  "getting-started": {
    "title": "入门指南",
    "description": "快速上手的步骤和说明"
  },
  "configuration": {
    "title": "配置",
    "description": "详细的配置选项和示例"
  }
}
```

## 分类示例

### 文档中心根目录

适用于 `src/content/docs/_meta.json`，管理顶级文档分类：

```json
{
  "official": {
    "title": "官方文档",
    "description": "关于本站的介绍、安装和配置指南"
  },
  "tutorials": {
    "title": "教程",
    "description": "详细的使用教程和示例"
  },
  "api": {
    "title": "API参考",
    "description": "开发者API接口文档和使用示例"
  },
  "plugins": {
    "title": "插件",
    "description": "可用插件列表和使用说明"
  },
  "best-practice": {
    "title": "最佳实践",
    "description": "使用本项目的推荐方法和技巧"
  }
}
```

### 分类目录

适用于特定分类目录，如 `src/content/docs/api/_meta.json`：

```json
{
  "overview": "概述",
  "authentication": "认证",
  "endpoints": {
    "title": "接口列表",
    "collapsed": false
  },
  "error-handling": "错误处理",
  "rate-limiting": "速率限制",
  "examples": {
    "title": "示例",
    "collapsed": true
  }
}
```

## 嵌套结构示例

### 两级嵌套

使用 `items` 属性创建嵌套结构：

```json
{
  "introduction": "介绍",
  "guides": {
    "title": "指南",
    "collapsed": false,
    "items": {
      "beginners": "初学者指南",
      "intermediate": "中级指南",
      "advanced": "高级指南"
    }
  },
  "api": {
    "title": "API",
    "collapsed": true,
    "items": [
      "authentication",
      "endpoints",
      "webhooks"
    ]
  }
}
```

### 三级嵌套

更复杂的嵌套结构：

```json
{
  "introduction": "介绍",
  "guides": {
    "title": "指南",
    "items": {
      "beginners": {
        "title": "初学者指南",
        "items": [
          "setup",
          "first-steps",
          "basic-concepts"
        ]
      },
      "advanced": {
        "title": "高级指南",
        "items": [
          "optimization",
          "security",
          "scaling"
        ]
      }
    }
  }
}
```

## 特殊功能示例

### 分隔符和外部链接

使用分隔符和外部链接：

```json
{
  "introduction": "介绍",
  "getting-started": "入门指南",
  "usage": "使用方法",
  
  "separator-1": {
    "title": "参考资料",
    "type": "separator"
  },
  
  "api-reference": "API参考",
  "configuration-reference": "配置参考",
  
  "separator-2": {
    "title": "外部资源",
    "type": "separator"
  },
  
  "github": {
    "title": "GitHub仓库",
    "href": "https://github.com/iflux-art/web"
  },
  "discord": {
    "title": "Discord社区",
    "href": "https://discord.example.com"
  }
}
```

### 隐藏项目

隐藏某些文档：

```json
{
  "public-doc": "公开文档",
  "internal-doc": {
    "title": "内部文档",
    "display": "hidden"
  },
  "beta-feature": {
    "title": "测试功能",
    "display": "hidden"
  }
}
```

## 总结

以上示例展示了 `_meta.json` 文件在不同场景下的配置方式。您可以根据自己的需求，选择合适的配置方式，或者组合多种配置方式，创建出最适合您项目的文档结构。

记住，好的文档结构应该是：
- 层次清晰，易于导航
- 分组合理，相关内容放在一起
- 命名一致，使用户容易理解
- 灵活可扩展，方便后续添加内容

通过合理使用 `_meta.json` 文件，您可以创建出既美观又实用的文档系统，提升用户的阅读体验。
