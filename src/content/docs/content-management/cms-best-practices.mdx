# 内容管理最佳实践

内容管理系统（CMS）是现代网站和应用程序的核心组件。本指南将介绍构建和维护高效内容管理系统的最佳实践。

## 第一部分：内容策略

### 1.1 内容架构设计

**层次化组织**：
```
content/
├── blog/                    # 博客文章
│   ├── 2024/               # 按年份组织
│   └── categories/         # 按分类组织
├── docs/                   # 文档系统
│   ├── guides/            # 指南
│   ├── tutorials/         # 教程
│   └── reference/         # 参考资料
└── pages/                  # 静态页面
    ├── about/
    └── contact/
```

**内容类型定义**：

### 1.2 内容生命周期管理

**内容状态流程**：
1. **草稿（Draft）** → 内容创建阶段
2. **审核（Review）** → 内容审查阶段
3. **发布（Published）** → 内容上线阶段
4. **更新（Updated）** → 内容维护阶段
5. **归档（Archived）** → 内容下线阶段

### 2.2 内容处理流程

**自动化内容处理**：
```javascript
// lib/content-processor.js
export class ContentProcessor {
  static async processMarkdown(content) {
    // 1. 解析 frontmatter
    const { data, content: body } = matter(content);
    
    // 2. 处理图片优化
    const optimizedContent = await this.optimizeImages(body);
    
    // 3. 生成目录
    const toc = await this.generateTOC(optimizedContent);
    
    // 4. SEO 优化
    const seoData = await this.generateSEO(data, optimizedContent);
    
    return {
      frontmatter: data,
      content: optimizedContent,
      toc,
      seo: seoData,
      readingTime: this.calculateReadingTime(optimizedContent)
    };
  }
  
  static calculateReadingTime(content) {
    const wordsPerMinute = 200;
    const words = content.split(/\s+/).length;
    return Math.ceil(words / wordsPerMinute);
  }
}
```

```javascript
// lib/cache.js
export class ContentCache {
  static async getContent(slug, options = {}) {
    const cacheKey = `content:${slug}`;
    
    // 尝试从缓存获取
    let content = await redis.get(cacheKey);
    
    if (!content || options.refresh) {
      // 重新生成内容
      content = await this.generateContent(slug);
      
      // 缓存内容（24小时）
      await redis.setex(cacheKey, 86400, JSON.stringify(content));
    }
    
    return JSON.parse(content);
  }
}
```

### 3.2 内容搜索优化

**全文搜索实现**：
```javascript
// lib/search.js
import { createIndex } from '@/lib/search-index';

export class ContentSearch {
  static async indexContent(content) {
    const index = createIndex();
    
    content.forEach(item => {
      index.add({
        id: item.slug,
        title: item.title,
        description: item.description,
        content: item.content,
        tags: item.tags,
        category: item.category
      });
    });
    
    return index;
  }
  
  static async search(query, options = {}) {
    const results = await this.index.search(query, {
      limit: options.limit || 10,
      fields: ['title', 'description', 'content'],
      boost: {
        title: 3,
        description: 2,
        content: 1
      }
    });
    
    return results;
  }
}
```

// scripts/content-quality-check.js
export async function checkContentQuality(filePath) {
  const content = await fs.readFile(filePath, 'utf8');
  const { data, content: body } = matter(content);
  
  const checks = {
    hasFrontmatter: !!data.title && !!data.description,
    hasValidDate: isValidDate(data.date),
    hasCategory: !!data.category,
    hasTags: Array.isArray(data.tags) && data.tags.length > 0,
    minWordCount: body.split(/\s+/).length >= 300,
    hasHeadings: /^#{1,6}\s/.test(body),
    hasCodeBlocks: /```/.test(body),
    validLinks: await checkLinks(body)
  };
  
  return checks;
}

```javascript
// lib/analytics.js
export class ContentAnalytics {
  static trackPageView(slug, metadata) {
    gtag('event', 'page_view', {
      page_title: metadata.title,
      page_location: window.location.href,
      content_group1: metadata.category,
      content_group2: metadata.tags?.join(',')
    });
  }
  
  static trackReadingProgress(slug, progress) {
    if (progress === 25 || progress === 50 || progress === 75 || progress === 100) {
      gtag('event', 'reading_progress', {
        event_category: 'Content',
        event_label: slug,
        value: progress
      });
    }
  }
}
```
