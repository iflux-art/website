# 业务组件

业务组件是项目中与具体功能模块紧密相关的组件，封装了特定的业务逻辑和UI展示。

## 博客业务组件

### 1. BlogCard (博客卡片)

用于在列表页展示文章摘要信息。

#### 基本用法

```tsx
import { BlogCard } from '@/features/blog/components/blog-card';

<BlogCard 
  post={{
    slug: 'getting-started-with-nextjs',
    title: 'Next.js入门指南',
    description: '从零开始学习Next.js框架',
    date: '2025-08-29',
    tags: ['Next.js', 'React', '教程']
  }}
/>
```

#### Props说明

| 属性名 | 类型 | 描述 |
|--------|------|------|
| post | BlogPost | 博客文章数据 |

#### 功能特性

- 显示文章标题和描述
- 展示发布日期和标签
- 提供到文章详情页的链接
- 响应式设计适配

### 2. TagCloudCard (标签云卡片)

展示所有文章标签的可视化组件。

#### 基本用法

```tsx
import { TagCloudCard } from '@/features/blog/components/tag-cloud-card';

<TagCloudCard 
  tags={[
    { name: 'Next.js', count: 5 },
    { name: 'React', count: 8 }
  ]}
/>
```

#### Props说明

| 属性名 | 类型 | 描述 |
|--------|------|------|
| tags | Tag[] | 标签数据列表 |

#### 功能特性

- 标签按使用频率大小显示
- 支持标签筛选功能
- 响应式设计适配
- 动画效果增强视觉体验

### 3. LatestPostsCard (最新文章卡片)

展示最近发布的文章列表。

#### 基本用法

```tsx
import { LatestPostsCard } from '@/features/blog/components/latest-posts-card';

<LatestPostsCard 
  posts={[
    {
      slug: 'getting-started-with-nextjs',
      title: 'Next.js入门指南',
      date: '2025-08-29'
    }
  ]}
/>
```

#### Props说明

| 属性名 | 类型 | 描述 |
|--------|------|------|
| posts | BlogPost[] | 文章数据列表 |

#### 功能特性

- 按发布时间排序显示
- 显示文章标题和日期
- 提供到文章详情页的链接
- 支持自定义显示数量

## 文档业务组件

### 1. DocsSidebar (文档侧边栏)

提供文档分类导航功能。

#### 基本用法

```tsx
import { DocsSidebar } from '@/features/docs/components/docs-sidebar';

<DocsSidebar 
  structure={[
    {
      slug: ['getting-started'],
      title: '快速开始',
      items: [
        {
          slug: ['getting-started', 'index'],
          title: '快速开始'
        }
      ]
    }
  ]}
/>
```

#### Props说明

| 属性名 | 类型 | 描述 |
|--------|------|------|
| structure | DocsStructure[] | 文档结构数据 |

#### 功能特性

- 自动生成导航结构
- 支持展开/折叠分类
- 高亮当前页面所在分类
- 响应式设计适配

### 2. DocContent (文档内容)

负责渲染文档内容。

#### 基本用法

```tsx
import { DocContent } from '@/features/docs/components/doc-content';

<DocContent 
  content="# 文档标题\n\n文档内容..."
  tableOfContents={[
    { id: 'introduction', text: '介绍', level: 1 }
  ]}
/>
```

#### Props说明

| 属性名 | 类型 | 描述 |
|--------|------|------|
| content | string | 文档内容（Markdown格式） |
| tableOfContents | TableOfContentsItem[] | 目录结构 |

#### 功能特性

- MDX内容解析和渲染
- 代码块高亮显示
- 图片和媒体内容处理
- 目录结构生成

## 搜索业务组件

### 1. SearchDialog (搜索对话框)

提供完整的搜索交互界面。

#### 基本用法

```tsx
import { SearchDialog } from '@/features/search/components/search-dialog';

<SearchDialog 
  open={isOpen}
  onOpenChange={setIsOpen}
/>
```

#### Props说明

| 属性名 | 类型 | 描述 |
|--------|------|------|
| open | boolean | 对话框是否打开 |
| onOpenChange | (open: boolean) => void | 对话框状态变化回调 |

#### 功能特性

- 模态对话框设计
- 搜索输入框
- 搜索结果展示区域
- 加载状态和错误处理

### 2. SearchResults (搜索结果)

展示搜索结果列表。

#### 基本用法

```tsx
import { SearchResults } from '@/features/search/components/search-results';

<SearchResults 
  results={[
    {
      type: 'blog',
      title: 'Next.js入门指南',
      description: '从零开始学习Next.js框架',
      path: '/blog/getting-started-with-nextjs'
    }
  ]}
/>
```

#### Props说明

| 属性名 | 类型 | 描述 |
|--------|------|------|
| results | SearchResult[] | 搜索结果列表 |

#### 功能特性

- 按内容类型分类展示
- 关键词高亮显示
- 结果摘要展示
- 响应式设计适配

## 组件设计原则

### 1. 业务逻辑封装

业务组件应该：

- 封装与业务相关的逻辑
- 提供清晰的接口
- 保持组件的可复用性
- 遵循单一职责原则

### 2. 数据流管理

- 通过Props接收数据
- 通过回调函数传递事件
- 合理使用状态管理
- 避免组件间直接数据依赖

### 3. UI一致性

- 遵循设计系统规范
- 保持与基础组件的一致性
- 提供统一的交互体验
- 支持主题定制

## 组件复用策略

### 1. 识别复用场景

在以下情况下考虑创建业务组件：

- 相同功能在多个页面使用
- 复杂业务逻辑需要统一实现
- 需要保持一致的UI展示

### 2. 抽象组件

创建业务组件时应：

- 分析可变部分和不变部分
- 设计合理的Props接口
- 提供默认行为和配置选项
- 添加充分的文档说明

### 3. 组件文档

每个业务组件都应包含：

- 清晰的功能说明
- Props类型定义和说明
- 使用示例
- 注意事项和限制

## 性能优化

### 1. 渲染优化

- 使用React.memo避免不必要的重渲染
- 合理使用useMemo和useCallback
- 懒加载非关键组件

### 2. 数据获取优化

- 使用TanStack Query管理数据获取
- 实现合理的缓存策略
- 避免重复数据请求

### 3. 打包优化

- 代码分割减少初始包大小
- Tree shaking移除未使用代码
- 按需加载组件和依赖

通过这些业务组件，项目能够构建出功能丰富、逻辑清晰且易于维护的用户界面。
