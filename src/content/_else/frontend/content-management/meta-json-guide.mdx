`_meta.json` 文件是一个强大的工具，用于自定义文档的结构、标题、描述和显示顺序。本指南将详细介绍如何使用 `_meta.json` 文件来管理您的文档集。

## 什么是 \_meta.json 文件？

`_meta.json` 是一个特殊的配置文件，可以放置在文档目录中，用于：

1. 自定义文档和目录的显示标题
2. 添加描述信息
3. 控制文档的显示顺序
4. 设置目录的折叠状态
5. 添加外部链接
6. 创建分隔符和特殊项目

## \_meta.json 文件的位置

`_meta.json` 文件可以放置在两个位置：

1. **根目录**：`src/content/docs/_meta.json`
   - 用于集中管理所有顶级文档集的标题、描述和顺序

2. **分类目录**：`src/content/docs/分类名称/_meta.json`
   - 用于管理特定分类内的文档和子目录

## 基本语法和结构

`_meta.json` 文件使用 JSON 格式，基本结构如下：

```json
{
  "文档ID或目录名": "显示标题",
  "另一个文档ID": {
    "title": "显示标题",
    "description": "描述文本"
  }
}
```

### 简单格式

最简单的格式是直接映射文档ID到显示标题：

```json
{
  "getting-started": "快速开始",
  "installation": "安装指南",
  "configuration": "配置说明"
}
```

### 完整格式

完整格式允许您添加更多元数据：

```json
{
  "getting-started": {
    "title": "快速开始",
    "description": "帮助您快速上手的入门指南"
  },
  "installation": {
    "title": "安装指南", 
    "description": "详细的安装步骤和环境配置"
  }
}
```

## 最佳实践

### 1. 命名规范

- 使用 kebab-case 命名文档ID
- 保持文档ID与文件名一致
- 使用有意义的描述性名称

```json
{
  "getting-started": "快速开始",
  "api-reference": "API 参考",
  "troubleshooting": "故障排除"
}
```

### 2. 描述编写

- 保持描述简洁明了
- 突出文档的核心价值
- 使用一致的语言风格

```json
{
  "installation": {
    "title": "安装指南",
    "description": "详细的安装步骤和环境配置要求"
  }
}
```

### 3. 顺序组织

- 按照学习路径组织文档顺序
- 将基础内容放在前面
- 将参考资料放在后面

```json
{
  "introduction": "介绍",
  "getting-started": "快速开始", 
  "tutorials": "教程",
  "guides": "指南",
  "api-reference": "API 参考",
  "troubleshooting": "故障排除"
}
```

## 自动化维护

### 1. 验证脚本

创建脚本来验证 `_meta.json` 文件的完整性：

```javascript
// scripts/validate-meta.js
const fs = require('fs');
const path = require('path');

function validateMetaJson(dirPath) {
  const metaPath = path.join(dirPath, '_meta.json');
  
  if (!fs.existsSync(metaPath)) {
    console.warn(`Missing _meta.json in ${dirPath}`);
    return;
  }
  
  const meta = JSON.parse(fs.readFileSync(metaPath, 'utf8'));
  const files = fs.readdirSync(dirPath)
    .filter(file => file.endsWith('.mdx'))
    .map(file => file.replace('.mdx', ''));
  
  // 检查是否所有文档都在 _meta.json 中
  files.forEach(file => {
    if (!meta[file]) {
      console.warn(`Missing entry for ${file} in ${metaPath}`);
    }
  });
  
  // 检查 _meta.json 中是否有不存在的文档
  Object.keys(meta).forEach(key => {
    if (!files.includes(key) && !key.startsWith('---')) {
      console.warn(`Entry ${key} in ${metaPath} has no corresponding file`);
    }
  });
}
```

### 2. 自动生成脚本

```javascript
// scripts/generate-meta.js
function generateMetaJson(dirPath) {
  const files = fs.readdirSync(dirPath)
    .filter(file => file.endsWith('.mdx'))
    .map(file => file.replace('.mdx', ''));
  
  const meta = {};
  files.forEach(file => {
    // 从文件名生成标题
    const title = file
      .split('-')
      .map(word => word.charAt(0).toUpperCase() + word.slice(1))
      .join(' ');
    
    meta[file] = title;
  });
  
  fs.writeFileSync(
    path.join(dirPath, '_meta.json'),
    JSON.stringify(meta, null, 2)
  );
}
```

## 常见问题

### Q: 如何处理中文文档名？

A: 建议文件名使用英文，在 `_meta.json` 中配置中文标题：

```json
{
  "getting-started": "快速开始",
  "installation-guide": "安装指南"
}
```

### Q: 如何组织多级目录？

A: 每个目录都应该有自己的 `_meta.json` 文件：

```
docs/
├── _meta.json
├── frontend/
│   ├── _meta.json
│   ├── basics/
│   │   ├── _meta.json
│   │   └── html.mdx
│   └── frameworks/
│       ├── _meta.json
│       └── react.mdx
```

### Q: 如何处理文档的版本控制？

A: 建议将 `_meta.json` 文件纳入版本控制，并在文档更新时同步更新元数据。

## 总结

`_meta.json` 文件是管理文档结构的强大工具。通过合理使用这个配置文件，您可以：

1. 创建清晰的文档导航结构
2. 提供有用的描述信息
3. 控制文档的显示顺序
4. 实现自动化的文档管理

记住要保持 `_meta.json` 文件与实际文档的同步，并定期验证配置的完整性。
