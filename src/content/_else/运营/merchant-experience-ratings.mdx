# 商品体验

|问题|计算公式|注意事项|运营策略|
|---|---|---|---|
|商品差评率|商品差评率 = 近30日商品差评订单数 / 近30日物流签收订单数|取用户首次评价结果|1. 关注商品差评数据，及时进行劣质货品汰换；<br>    <br>2. 加强品控保证商品质量，为用户提供优质产品；<br>    <br>3. 商品宣传符合实际，不进行虚假、夸大宣传，避免消费者产生错误认知；<br>    <br>4. 如遇用户恶意评价导致商品差评率数据异常，商家可在抖店后台-店铺-举报中心发起用户举报，举报通过订单不参与相应指标考核。|
|商品品质退货率|商品品质退货率=近30日产生商品品退的订单/近30日物流签收订单量|取用户首次申请售后原因|1. 加强品控保证商品质量，为用户提供优质产品；<br>    <br>2. 保证发货运输质量，避免出现少件破损问题；<br>    <br>3. 商品宣传符合实际，不进行虚假、夸大宣传，避免消费者产生错误认知；<br>    <br>4. 开通正品保障等特色服务可增强用户正品认知；<br>    <br>5. 当前品质退货判断仅考虑首次品退，需前置做好退换条件说明及售后服务咨询；<br>    <br>6. 如遇用户恶意售后行为导致品质退货数据异常，商家可在抖店后台-店铺-举报中心发起用户举报，举报通过订单不参与相应指标考核。|

# 物流体验

|问题|计算公式|注意事项|运营策略|
|---|---|---|---|
|订单配送时长|订单配送时长 =全部签收订单配送时长/签收订单量|订单配送时长是指订单揽收到物流签收所用时长|1. 物流跟踪：及时跟踪订单配送情况，对平台给出的中转超时、配送超时等预警订单及时处理；<br>    <br>2. 异常报备：如出现不可抗原因导致物流履约问题，提前通过抖店后台-店铺-店铺保障-报备中心进行报备。|
|24小时支付-揽收率|24小时支付-揽收率 = 近30天支付订单中支揽在24小时内的现货订单数 / 近30天应揽收现货订单数|无需发货订单、定制类订单、跨境BC&CC订单不参与计算，BIC订单揽收时长为入仓提交时间-支付时间，BTAS订单揽收时长为卖家给质检机构发货时间-支付时间|1. 库存盘点：定期对商品库存进行盘点，并及时将库存数据同步店铺运营，确保商品系统库存与实际库存一致，避免超卖缺货；<br>    <br>2. 物流优化：选择平台推荐物流供应商作，保证用户体验；<br>    <br>3. 物流跟踪：加深与物流公司合作机制，每天上门取件，及时跟踪订单揽收情况，对平台揽收异常预警的订单及时处理；<br>    <br>4. 异常报备：如出现不可抗原因导致物流履约问题，提前通过抖店后台-店铺-店铺保障-报备中心进行报备。|
|发货问题负向反馈率|发货问题负向反馈率 = 近30日支付中产生发货问题负反馈的订单数 / 近30日支付订单数|现货时效内【发货慢】的负反馈不计入考核|1. 发货模式优化：降低预售商品比例，可有效减少用户发货速度相关的负反馈；<br>    <br>2. 包装改善：改善商品的包装，减少商品在物流运送中的破损，提升用户感官体验，减少负反馈；<br>    <br>3. 异常报备：如出现不可抗原因导致物流履约问题，提前通过抖店后台-店铺-店铺保障-报备中心进行报备，同时与消费者做好沟通和补偿。|

# 服务体验

|问题|计算公式|注意事项|运营策略|
|---|---|---|---|
|售后拒绝率|售后拒绝率 = 近30日被商家拒绝的发货后售后申请单量 / 近30日发货后售后申请单总量|不支持7天无理由的商品只考核发货后商品品质原因售后拒绝的订单|1. 同意退货前，可在详情页/退货地址/快递包裹里附上“退换货须知”，避免不符合退货条件导致拒绝；<br>    <br>2. 同意退货后，可提醒消费者及时上传正确退货物流单号，避免错误物流信息导致拒绝；<br>    <br>3. 若售后处理时效即将截止，还未收到退货商品，建议使用【延长收货】功能，同时与消费者积极沟通协商，协商一致可引导消费者暂时撤销售后申请；<br>    <br>4. 针对消费者售后申请应积极主动与消费者沟通协商，协商一致可引导消费者修改或取消售后申请，切记不要贸然拒绝消费者。若核实是快递责任，可进一步引导消费者联系快递方索赔。[用户修改售后信息使用说明](https://school.jinritemai.com/doudian/web/article/106724?rank=0&fromPage=search_result&searchInfo=12285814664860500178%3A0%3A0&query=%E5%A6%82%E4%BD%95%E4%BF%AE%E6%94%B9%E5%94%AE%E5%90%8E&resultType=%E5%8A%9F%E8%83%BD)|
|平台求助率|平台求助率=近30日支付订单中产生投诉或纠纷商责的订单数 / 近30日支付订单数||1. 积极配合有效举证：对于进入平台判决的订单，请积极配合提供真实有效举证资料，并配合平台调解；<br>    <br>2. 及时响应消费者售后诉求：可通过培训和考试，提升客服人员的售后处理专业能力，快速响应消费者的售后问题，及时处理退款/退货申请；<br>    <br>3. 用户体验受损补偿：对于商家原因或不可抗力原因导致订单履约异常用户体验受损的情况，第一时间制定主动告知和补偿方案，避免用户体验受损升级投诉；<br>    <br>4. 开通特色权益：开通运费险、坏单包赔、过敏险等特色服务保护消费者权益。|
|IM不满意率|IM不满意率=近30日IM差评（1-3星）数 / 近30日有评价IM数|只考核发起时间在8:00:00-22:59:59期间的人工客服会话，30天内评价数不足10个的商家不计算该指标|1. 客服上岗培训：客服上岗前需通过培训和考试，确保客服熟练掌握商品、活动、服务政策、系统功能等内容，快速准确的服务用户；<br>    <br>2. 客服绩效考核：设置合理的绩效考核制度，可督促客服高效服务用户；<br>    <br>3. 用户体验受损补偿：对于商家原因或不可抗力原因导致订单履约异常用户体验受损的情况，第一时间制定主动告知和补偿方案，降低用户体验受损评价不满意的几率；<br>    <br>4. 售后权益开通：开通极速退、运费险等服务，保证用户售后体验。|
|退货退款自主完结时长|退货退款自主完结时长=近30天每条退货退款（含换货）售后单中等待商家操作的时间总和/近30天退货退款（含换货）售后单量||1. 提高售后工具使用效率：合理配置使用售后小助手推荐策略，提升售后处理时效；<br>    <br>2. 提高客服专业能力：可通过培训和考试，提升客服人员的售后处理专业能力，快速响应消费者的售后问题，及时处理退款/退货申请；<br>    <br>3. 拒绝售后要合规：拒绝售后要符合平台售后规则，减少不必要的拒绝；<br>    <br>4. 如实描述商品：不夸大宣传、不虚假宣传，减少因描述不符产生的售后沟通及退款处理；<br>    <br>5. 开通售后权益：开通极速退等特色服务，保证用户售后时效体验。|
|仅退款自主完结时长|仅退款自主完结时长=仅退款售后单等待商家操作时间总和/仅退款售后单量||1. 提高售后工具使用效率：合理配置使用售后小助手推荐策略，提升售后处理时效；<br>    <br>2. 提高客服专业能力：可通过培训和考试，提升客服人员的售后处理专业能力，快速响应消费者的售后问题，及时处理退款/退货申请；<br>    <br>3. 拒绝售后要合规：拒绝售后要符合平台售后规则，减少不必要的拒绝；<br>    <br>4. 如实描述商品：不夸大宣传、不虚假宣传，减少因描述不符产生的售后沟通及退款处理；<br>    <br>5. 开通售后权益：开通极速退等特色服务，保证用户售后时效体验。|
|IM平均响应时长|IM平均响应时长=近30日工作时间消费者与商家IM对话回复时长之和 / 近30日工作时间人工咨询对话轮次总数|只考核发起时间在8:00:00-22:59:59期间的人工客服会话|1. 人力预测和排班优化：根据订单量和历史咨询人次进行来访量预测，根据时段来访预估可以分早班、晚班、半班等多种形式；<br>    <br>2. 客服权限开通：店铺设置专职客服或多客服情况下，必须为客服员工开通子账号，通过多账号进行买家接待回复和售后处理等业务，避免使用主账号登录造成回复遗漏等情况；<br>    <br>3. 客服接待上限设置：按照每个客服能力设置合适的接待上限，避免客服同时接待多个用户无法及时回复；<br>    <br>4. 大促应急预案：根据大促活动计划制定人力助接计划，安排兼职客服或其他职能人员进行接待。|
